**[时长 / Duration: 20 秒]**

任务是：利用加州人口普查数据，预测每个街区的房屋中位价。数据集来自 sklearn 的 `fetch_california_housing`，包含 20,640 条样本，8 个原始特征——收入中位数、房龄、房间数、卧室数、人口、住户数、纬度和经度。目标值是房屋中位价，单位是十万美元。

> The task: predict the median house value for each block using California census data. The dataset comes from sklearn's `fetch_california_housing`, with 20,640 samples and 8 original features — MedInc, HouseAge, AveRooms, AveBedrms, Population, AveOccup, Latitude, and Longitude. ---

**[时长 / Duration: 25 秒]**

项目按作业要求分为三个层次：基础层用 NumPy 实现 MSE 和 MAE 指标，训练线性回归和随机森林作为基线；进阶层对比两种特征选择方法，并实现纯 NumPy 的加权模型融合；扩展层做了 10×10 地理热力图和 IQR 极端值分析，展示模型在地理空间上的表现差异。

> The project is organized into three layers as required: the Basic layer implements MSE and MAE in pure NumPy, with Linear Regression and Random Forest as baselines; the Advanced layer compares two feature selection methods and implements weighted model fusion in pure NumPy; the Extended layer features a 10×10 geospatial heatmap and IQR-based outlier analysis, revealing how model performance varies geographically.

---

**[时长 / Duration: 30 秒]**

我的四个关键设计选择：第一，预处理使用中位数填补缺失值、StandardScaler 标准化，同时手动构造了两个比值特征——户均房间数和卧室占比，将特征从 8 维扩展到 10 维。第二，训练测试集按 80/20 划分，随机种子固定为 42，确保结果可复现。第三，融合权重根据各模型单独表现分配——随机森林全特征最优得 0.35，两个特征选择的 RF 各得 0.275，线性回归因线性假设限制仅得 0.10。第四，极端值检测用 IQR 乘以 1.5 的阈值，比 Z-score 更适合有偏分布。

> My four key design choices: First, preprocessing uses median imputation for missing values and StandardScaler for normalization, plus two engineered ratio features — RoomsPerHousehold and BedroomsPerRoom — expanding features from 8 to 10 dimensions. Second, an 80/20 train-test split with random seed fixed at 42 ensures full reproducibility. Third, fusion weights are assigned based on individual model performance — RF with all features performs best at 0.35, the two feature-selected RFs each get 0.275, and Linear Regression gets only 0.10 due to its linear assumption limitations. Fourth, outlier detection uses IQR × 1.5 rather than Z-score, as it's more suitable for skewed distributions.

### 2.1 Basic Layer — 数据处理与基线模型 / Data Processing & Baseline Models（约 1 分 30 秒）

**[时长 / Duration: 25 秒]**
`[展示屏幕 / Show on screen: data_prep.py — load_data() function code and terminal output]`

首先看数据加载。我用 `fetch_california_housing()` 获取数据，装入 Pandas DataFrame，然后构造两个新特征：`RoomsPerHousehold` 等于平均房间数除以住户数，`BedroomsPerRoom` 等于平均卧室数除以房间数。最后用 `.to_numpy()` 转为 NumPy 数组——X 的 shape 是 20640 行 × 10 列，y 是 20640 个目标值。

> First, data loading. I use `fetch_california_housing()` to obtain the data, load it into a Pandas DataFrame, then engineer two new features: `RoomsPerHousehold` equals AveRooms divided by AveOccup, and `BedroomsPerRoom` equals AveBedrms divided by AveRooms. Finally, `.to_numpy()` converts everything to NumPy arrays — X has shape 20640 rows × 10 columns, y has 20640 target values.

---

**[时长 / Duration: 25 秒]**
`[展示屏幕 / Show on screen: fill_missing_values() and standardize_features() code]`

预处理分两步：缺失值填充和标准化。缺失值用的是 NumPy 的 `np.nanmedian` 计算各列中位数填充，而非 sklearn 的 SimpleImputer——这是纯 NumPy 实现。标准化使用 sklearn 的 `StandardScaler`，使所有特征均值为 0、标准差为 1。

> Preprocessing has two steps: missing value imputation and standardization. Missing values are filled using NumPy's `np.nanmedian` to compute column medians — not sklearn's SimpleImputer — this is a pure NumPy implementation. Standardization uses sklearn's `StandardScaler`, centering all features to mean 0 and standard deviation 1.

---

**[时长 / Duration: 20 秒]**
`[展示屏幕 / Show on screen: LinearRegression training code and terminal coefficient table]`

训练线性回归模型。这是我们的基线。注意屏幕上的系数：`MedInc` 的系数是 0.79——收入中位数每增加一个标准差，房价就上涨约 7.9 万美元。`Latitude` 和 `Longitude` 的系数分别为 -0.89 和 -0.83，说明地理位置是房价的关键驱动因素。这为后续的地理分析埋下了伏笔。

> Training the Linear Regression model — our baseline. Note the coefficients on screen: `MedInc` has a coefficient of 0.79 — for every standard deviation increase in median income, house value rises by about $79,000. `Latitude` and `Longitude` have coefficients of -0.89 and -0.83 respectively, showing that geographic location is a key driver of house prices. This foreshadows our later geospatial analysis.

---

**[时长 / Duration: 20 秒]**
`[展示屏幕 / Show on screen: evaluation.py — compute_mse() and compute_mae() core code, plus terminal MSE/MAE output]`

指标计算全部用 NumPy 手动实现。MSE 就是预测值和真实值之差的平方的均值，MAE 是绝对差的均值——两行代码就完成了。线性回归的 MSE 是 0.4561，MAE 为 0.4862，R² 为 0.6519。作为基线，这为后面的模型改进提供了参照。

> All metrics are computed manually with NumPy. MSE is simply the mean of squared differences between predictions and true values; MAE is the mean of absolute differences — just two lines of code each. Linear Regression achieves an MSE of 0.4561, MAE of 0.4862, and R² of 0.6519. As a baseline, this provides a reference point for all subsequent model improvements.

### 2.2 Advanced Layer — 特征选择与模型融合 / Feature Selection & Model Fusion（约 2 分 15 秒）

**[时长 / Duration: 25 秒]**
`[展示屏幕 / Show on screen: feature_selection.py — pearson_correlation_numpy() for-loop code, plus terminal Top-5 feature names and correlation scores]`

进阶层首先做特征选择。方法一是纯 NumPy 的 Pearson 相关系数——对每个特征，手动计算它与目标值的协方差除以标准差之积。注意代码中先减去均值再求内积，完全不用 sklearn。选出 |r| 最大的 5 个特征。

> The Advanced layer begins with feature selection. Method one is pure NumPy Pearson correlation — for each feature, manually computing the covariance with the target divided by the product of their standard deviations. Notice the code subtracts the mean first, then computes the dot product — entirely without sklearn. We select the top 5 features by absolute correlation.

---

**[时长 / Duration: 15 秒]**
`[展示屏幕 / Show on screen: select_features_sklearn_freg() calling SelectKBest code, plus terminal F-score output]`

方法二用 sklearn 的 `SelectKBest` 配合 F 回归检验，同样选 Top 5。终端展示了 F 分数。

> Method two uses sklearn's `SelectKBest` with F-regression scoring, also selecting the top 5. The terminal displays the F-scores.

---

**[时长 / Duration: 15 秒]**
`[展示屏幕 / Show on screen: compare_feature_sets() output — intersection and difference lists]`

对比两种方法的结果。在 random_state=42 下，两者选出了完全一致的五维特征：MedInc、BedroomsPerRoom、RoomsPerHousehold、AveRooms 和 Latitude。交集为 5/5——这是因为 Pearson 相关系数和 F-regression 虽然数学原理不同，但在强线性相关特征上排序高度一致。

> Comparing the two methods. Under random_state=42, both selected exactly the same five features: MedInc, BedroomsPerRoom, RoomsPerHousehold, AveRooms, and Latitude — a perfect 5/5 overlap. This shows that Pearson correlation and F-regression, despite their different mathematical foundations, produce highly consistent rankings on strongly linear features.

---

**[时长 / Duration: 25 秒]**
`[展示屏幕 / Show on screen: models.py — train_random_forest() hyperparameters, plus terminal MSE output for each RF model]`

接下来训练随机森林。超参数：300 棵树、最大深度 20、使用全部 CPU 核心并行。分别在全特征、Pearson 特征和 F-Reg 特征上各训练一个模型。看终端输出：随机森林全特征 MSE 为 0.2531，比线性回归降低了约 44%，R² 从 0.65 提升到 0.81——非线性模型的优势非常明显。

> Next, training Random Forest. Hyperparameters: 300 trees, max depth 20, using all CPU cores in parallel. We train one model on all features, one on Pearson-selected features, and one on F-Reg-selected features. Looking at the terminal: RF with all features achieves MSE of 0.2531 — roughly a 44% reduction from Linear Regression — and R² improves from 0.65 to 0.81. The advantage of a non-linear model is striking.

---

**[时长 / Duration: 15 秒]**
`[展示屏幕 / Show on screen: output/feature_selection_mse.png bar chart]`

这是特征选择方法的 MSE 对比柱状图。两组结果完全一致——MSE 都是 0.4004。两个方法选出了相同的特征子集，所以模型表现自然相同。这意味着对于本数据集，Pearson 和 F-regression 在 Top-5 选择上可以互相替代。

> Here is the MSE comparison bar chart for the two feature selection methods. The results are identical — both yield an MSE of 0.4004. Since both methods selected the same feature subset, the model performance is naturally identical. This implies that for this dataset, Pearson and F-regression are interchangeable for top-5 selection.

---

**[时长 / Duration: 35 秒]**
`[展示屏幕 / Show on screen: model_fusion.py — train_and_fuse_models() core implementation, plus terminal fusion model MSE/MAE comparison table]`

最后是模型融合。`train_and_fuse_models` 是融合的核心函数——自动执行特征选择、训练四个基模型（线性回归和三种随机森林），然后用 NumPy 对预测值做加权求和。权重配置：线性回归 0.10、RF 全特征 0.35、两个特征选择 RF 各 0.275。这个分配基于各模型单独表现：RF 全特征 MSE 最低所以权重最高；两组特征选择 RF 性能持平所以均分。注意权重会被归一化，最终和为 1。

融合模型的 MSE 为 0.3167，R² 为 0.7583。融合优于线性回归和两个特征选择 RF，但不及全特征 RF（MSE 0.2531）——这说明当单一强模型已经充分利用了全部特征信息时，简单的加权平均融合不一定能带来额外收益。融合的优势更多体现在：当各个子模型使用不同特征子集或不同类型算法时，可以整合互补信息。

> Finally, model fusion. `train_and_fuse_models` is the core fusion function — it automatically performs feature selection, trains four base models (Linear Regression and three Random Forest variants), then uses NumPy to compute the weighted sum of predictions. The weight configuration: Linear Regression 0.10, RF all-features 0.35, and each feature-selected RF 0.275. This allocation is based on individual performance: RF all-features has the lowest MSE so gets the highest weight; the two feature-selected RFs perform similarly so share equal weight. Note that weights are normalized to sum to 1.

> The fusion model achieves an MSE of 0.3167 and R² of 0.7583. It outperforms Linear Regression and both feature-selected RFs, but doesn't surpass the full-feature RF (MSE 0.2531) — demonstrating that when a single strong model already leverages all feature information, simple weighted averaging may not yield additional gains. Fusion's strength lies more in integrating complementary information when sub-models use different feature subsets or algorithm types.

### 2.3 Extended Layer — 地理热力图与极端值分析 / Geospatial Heatmap & Outlier Analysis（约 1 分 45 秒）

**[时长 / Duration: 30 秒]**
`[展示屏幕 / Show on screen: visualization.py — plot_geospatial_heatmap() grid construction code, plus output/geospatial_heatmap.png]`

扩展层首先做地理空间分析。核心逻辑：将测试集样本按经纬度划分成 10×10 网格。用 `np.linspace` 生成等间距边界，然后双层 for 循环遍历每个格子，计算落在该格子内样本的平均绝对预测误差。为什么是 10×10？网格太细每个格子样本太少、统计不稳定，网格太粗又失去空间分辨率——10×10 是折中方案。

> The Extended layer begins with geospatial analysis. The core logic: divide test-set samples into a 10×10 grid by latitude and longitude. We use `np.linspace` to generate evenly-spaced bin boundaries, then a nested for-loop iterates over each cell, computing the mean absolute prediction error for samples falling within it. Why 10×10? Too fine a grid means too few samples per cell and unstable statistics; too coarse a grid loses spatial resolution — 10×10 is the balanced trade-off.

---

**[时长 / Duration: 25 秒]**
`[展示屏幕 / Show on screen: switch back to heatmap, use mouse to point out high-error regions]`

看生成的热力图。色阶从绿色（低误差）到红色（高误差）。可以看到沿海区域和洛杉矶、旧金山等大都市区的误差偏高——这些地区的房价波动更大、极端值更多，模型难以精确拟合。内陆地区的误差普遍较低。这说明地理位置确实是房价预测误差的关键因素。

> Looking at the heatmap. The color scale ranges from green (low error) to red (high error). We can see that coastal areas and major metro regions like Los Angeles and San Francisco have higher errors — these areas experience greater price volatility and more extreme values, making them harder for the model to fit accurately. Inland regions generally show lower errors. This confirms that geographic location is indeed a key factor in prediction error.

---

**[时长 / Duration: 15 秒]**
`[展示屏幕 / Show on screen: analysis.py — detect_outliers_iqr() code]`

接下来是极端值分析。检测方法选 IQR 而非 Z-score——因为房价数据明显右偏，Z-score 假设正态分布会误判。Q1 减 1.5 倍 IQR 作为下界，Q3 加 1.5 倍 IQR 作为上界，目标值超出这个范围的标记为极端值。

> Next, outlier analysis. We choose IQR over Z-score for detection — because house price data is clearly right-skewed, and Z-score's normality assumption would produce false positives. We set the lower bound at Q1 minus 1.5× IQR and the upper bound at Q3 plus 1.5× IQR; targets falling outside this range are flagged as outliers.

---

**[时长 / Duration: 15 秒]**
`[展示屏幕 / Show on screen: terminal outlier analysis output — percentage, target range, feature mean comparison table]`

从输出可以看到：极端值共 1,071 条，占比 5.19%，目标值范围集中在 4.83 到 5.00（即 48 万到 50 万美元，触及数据集上限）。极端样本的收入中位数平均为 7.62，是正常样本（3.67）的两倍多；户均房间数也显著更高。这些"豪宅"拉高了整体预测难度。

> From the output: 1,071 outliers detected, accounting for 5.19% of the data. Their target values cluster between 4.83 and 5.00 (i.e., $483K–$500K, hitting the dataset's upper cap). Outlier samples have a mean MedInc of 7.62 — more than double the normal mean of 3.67 — and significantly more rooms per household. These "luxury homes" increase the overall prediction difficulty.

---

**[时长 / Duration: 25 秒]**
`[展示屏幕 / Show on screen: output/outlier_removal_comparison.png comparison bar chart]`

移除极端值后重新训练最优模型。这里的最优模型是根据 MSE 从基础层和进阶层所有模型中自动选出的——本运行中为全特征随机森林。对比柱状图展示了移除前后的 MSE 和 MAE：MSE 从 0.2531 降至 0.1988，降低 21.4%；MAE 从 0.3276 降至 0.2972，降低 9.3%。性能提升明显。这证实极端高房价样本是模型误差的主要来源。移除它们后，模型在大多数"正常"房屋上的预测更加精准。

> After removing outliers, we retrain the best model — automatically selected from all Basic and Advanced layer models by lowest MSE; in this run, it's RF with all features. The comparison bar chart shows MSE and MAE before and after removal: MSE drops from 0.2531 to 0.1988 (a 21.4% reduction), and MAE drops from 0.3276 to 0.2972 (a 9.3% reduction) — a clear performance improvement. This confirms that extreme high-price samples are the main source of model error. After removing them, the model predicts more accurately on the majority of "normal" homes.
